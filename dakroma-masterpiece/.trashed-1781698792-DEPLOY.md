# 📱 DAKROMA MASTER PIECE — ANDROID DEPLOYMENT GUIDE

Deploy your game online from your Android phone using free tools.

---

## 🛠️ Apps You Need

| App | Purpose | Download |
|-----|---------|----------|
| **Termux** | Run Node.js locally | F-Droid or Play Store |
| **ZArchiver** | Unzip the project | Play Store |
| **GitHub app** (or Chrome) | Upload code | Play Store |
| **Chrome** | Access Render + Netlify | Pre-installed |

---

## STEP 1 — Test Locally in Termux

```bash
# Install Node.js in Termux (first time only)
pkg update && pkg upgrade -y
pkg install nodejs -y
pkg install git -y

# Unzip and run
cd ~/storage/downloads
unzip dakroma-masterpiece.zip
cd dakroma-masterpiece
npm install
node server.js
```

Open Chrome → `http://localhost:3000` ✅

---

## STEP 2 — Upload to GitHub

### Option A: GitHub Website (Chrome)
1. Go to https://github.com → Sign in
2. Click **New repository** → Name it `dakroma-masterpiece` → Public → Create
3. Click **uploading an existing file**
4. Drag and drop all files from ZArchiver
5. Click **Commit changes**

### Option B: Termux Git
```bash
cd ~/storage/downloads/dakroma-masterpiece
git init
git add .
git commit -m "Initial commit 💜"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/dakroma-masterpiece.git
git push -u origin main
```

---

## STEP 3 — Deploy Backend to Render (Free)

1. Go to https://render.com → Sign up with GitHub
2. Click **New** → **Web Service**
3. Connect your `dakroma-masterpiece` repo
4. Fill in:
   - **Name:** dakroma-masterpiece
   - **Runtime:** Node
   - **Build Command:** `npm install`
   - **Start Command:** `node server.js`
   - **Plan:** Free
5. Click **Create Web Service**
6. Wait ~2 minutes → you get a URL like:
   ```
   https://dakroma-masterpiece.onrender.com
   ```
7. ✅ Copy this URL — you need it next!

---

## STEP 4 — Update Your Render URL in game.js

Open `public/game.js` and find this line near the top:

```javascript
const SOCKET_URL =
  window.location.hostname === 'localhost' ||
  window.location.hostname === '127.0.0.1'
    ? undefined
    : 'https://YOUR-RENDER-BACKEND.onrender.com';  // ← CHANGE THIS
```

Replace `https://YOUR-RENDER-BACKEND.onrender.com` with your actual Render URL.

Then push to GitHub:
```bash
git add public/game.js
git commit -m "Update Render URL"
git push
```

---

## STEP 5 — Deploy Frontend to Netlify (Free)

1. Go to https://netlify.com → Sign up
2. Click **Add new site** → **Deploy manually**
3. Drag and drop ONLY the **`public/`** folder
4. Netlify gives you a URL like:
   ```
   https://dakroma-xyz123.netlify.app
   ```
5. Share this link with your friends! 🎉

### Or connect GitHub for auto-deploy:
1. **Add new site** → **Import from Git**
2. Choose your repo
3. Set:
   - **Base directory:** `public`
   - **Publish directory:** `public`
4. Deploy ✅

## 🔄 Updating the Game

After making changes:

```bash
git add .
git commit -m "Update game"
git push
```

- Render auto-redeploys the backend
- Netlify auto-redeploys the frontend
- No manual steps needed ✅

---

## ⚠️ Important Notes

1. **Render free tier sleeps** after 15 min of no traffic.
   First player to connect may wait ~30 seconds.
   Upgrade to paid plan to avoid this.

2. **names.json** is stored on Render's disk.
   On Render free tier, the disk resets on redeploy.
   To persist data, add a Render Disk or use a database.

3. **CORS is open** (`origin: '*'`) for development.
   For production, change to your Netlify domain.

## 🆘 Troubleshooting

| Problem | Fix |
|---------|-----|
| `npm: not found` in Termux | Run `pkg install nodejs` |
| Can't connect online | Check Render URL in game.js |
| White screen | Open Chrome DevTools → check console |
| Name won't save | Username must be 2-20 characters |
| Render keeps sleeping | Upgrade plan or use cron to ping |
